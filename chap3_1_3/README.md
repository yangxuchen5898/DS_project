# chap3_1_3

在顺序表中设计函数实现以下操作：

在一个顺序表中如果一个数据值有重复出现，则留下第一个这样的数据值，并删除其他所有重复的元素，使表中所有元素的值均不相同。

## 思路（by chatgpt）：

从前往后扫描顺序表，对每个元素，检查它在前面是否已经出现过。

具体做法是：

1. 从第二个元素开始往后看。
2. 对当前元素，向前逐个比较它前面的所有元素。
3. 如果前面已经有和它相同的值，说明它是重复元素，应该删除。
4. 如果前面没有相同值，就保留。
5. 一直处理到表尾。

核心思想就是：

**“保留第一次出现的，后面再遇到相同值就删掉。”**

删除时在顺序表中通常要把后面的元素整体前移一位，并把表长减 1。
因为删除后当前位置会换上一个新元素，所以这一位要继续检查，不能直接跳到下一个位置。

## 思路（by gemini）：

实现这个操作主要有两种思路，取决于你更倾向于节省内存空间，还是追求更快的运行速度。

### 思路一：双重循环对比（时间换空间，最常规考法）

不需要申请额外的存储空间，直接在原表上进行操作。

1. **外层遍历锁定基准**：用一个外层循环（指针 `i`）从前向后遍历顺序表，每次固定一个元素作为“参照物”。
2. **内层遍历查找重复**：用一个内层循环（指针 `j`）从 `i` 的下一个位置开始，一直扫描到表尾。
3. **覆盖删除**：如果发现 `j` 指向的元素与 `i` 指向的元素值相同，说明出现了重复。此时执行顺序表的删除操作：将 `j` 位置之后的所有元素统一向前移动一位，覆盖掉这个重复值，并将顺序表的总长度减 1。
4. **关键细节**：一旦执行了覆盖删除，原本排在后面的元素会移到当前 `j` 的位置。因此，删除操作后 `j` 指针**不能前进**，必须停在原地继续检查新挪过来的元素。

### 思路二：借助哈希集合/标志数组（空间换时间，效率更高）

需要额外申请一块内存来记录某个值是否已经出现过。

1. **准备辅助工具**：创建一个空的哈希集合（或大小合适的布尔数组），用来登记出现过的数据。同时准备一个“写入指针”（初始指向上标 0）。
2. **单次遍历**：从头到尾遍历顺序表中的每一个元素。
3. **检查与重写**：

   * 如果当前元素在集合中**没有记录**：说明是第一次出现。把它登记进集合，并将该元素写入到“写入指针”所在的位置，然后“写入指针”向后移动一位。

   * 如果当前元素在集合中**已有记录**：说明是重复值，直接跳过不作处理。


4. **更新表长**：遍历结束后，直接将顺序表的总长度修改为“写入指针”当前的值即可。
